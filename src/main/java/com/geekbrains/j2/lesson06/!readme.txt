1. Если сервер не запущен, клиент будет пытаться подключиться с задержкой в 2 секунды и выводить предупреждения.
2. Сообщения в окно чата приходят только с сервера, т.е. при отправке сообщения из чата, оно идёт на сервер,
а уже от него попадает в текстовое поле.
3. BufferedReader работает в отдельном потоке и при получении не пустой строки, кидает её в чат.
4. При отправке с клиента команды /q, сервер гасится, так же как если отправить команду из консоли самого сервера.
Сам клиент продолжает работать - выводится информационное сообщение.
